# Portfolio Tracker

เว็บติดตามพอร์ตการลงทุนส่วนตัว สำหรับหุ้นและกองทุนทั้งไทยและต่างประเทศ —
แสดงยอดถือครองพร้อมกำไร/ขาดทุนแบบสด และคำนวณ rebalancing (ต้องซื้อ/ขายเท่าไหร่
ให้ได้สัดส่วนที่ต้องการ) เน้นตลาดไทย (SET/mai, THB, รองรับ RMF/SSF/ThaiESG)

---

## โปรเจกต์นี้ทำอะไร

เครื่องมือสำหรับคนที่ถือหุ้น/กองทุนหลายตัวหลายพอร์ต แล้วอยากรู้ว่า:
- ตอนนี้ถืออะไรอยู่บ้าง มูลค่าเท่าไหร่ กำไร/ขาดทุนเท่าไหร่
- สัดส่วนพอร์ตตอนนี้เบี่ยงจากที่ตั้งเป้าไว้แค่ไหน และต้องซื้อ/ขายอะไรกี่บาท
  เพื่อกลับเข้าสัดส่วนเป้าหมาย

### ความสามารถปัจจุบัน
- **หลายพอร์ต** — แยกพอร์ตได้ตามต้องการ (เช่น พอร์ตเกษียณ, พอร์ตเก็งกำไร)
- **ยอดถือครอง + กำไร/ขาดทุน** — คำนวณจากประวัติการซื้อขาย ไม่ใช่ตัวเลขที่พิมพ์เอง
- **กรอกราคาเอง** — ใส่ราคา/NAV ล่าสุดเพื่อคำนวณมูลค่าปัจจุบัน
- **ตั้งสัดส่วนเป้าหมาย + rebalancing** — ตั้ง target % ต่อสินทรัพย์ แล้วระบบบอก
  ว่าแต่ละตัว drift ไปเท่าไหร่ และต้องซื้อ/ขายกี่บาทกี่หน่วย

ดูแผนงานที่จะทำต่อได้ใน `ROADMAP.md`

---

## แนวคิดหลักเบื้องหลัง (สำคัญ)

**ยอดถือครองไม่ได้ถูกเก็บตรงๆ แต่คำนวณจากประวัติการซื้อขาย (transactions)**

แทนที่จะพิมพ์ว่า "ถือ PTT 100 หุ้น" ระบบจะเก็บ "รายการซื้อขาย" ทุกครั้ง แล้ว
คำนวณยอดถือครอง ต้นทุนเฉลี่ย และกำไร/ขาดทุนออกมาเอง เหตุผลคือวิธีนี้ให้
ต้นทุนเฉลี่ยที่ถูกต้อง คำนวณกำไรตอนขายได้ และรองรับปันผล/DCA ได้ ซึ่งการเก็บ
แค่ "ยอดปัจจุบัน" ทำไม่ได้ (อ่านเหตุผลเต็มใน `DECISIONS.md` D1-D6)

## Tech stack

| ส่วน | ใช้อะไร | ทำไม |
|------|---------|------|
| Frontend | Next.js (App Router) + TypeScript | React framework ยอดนิยม, deploy ฟรีบน Vercel |
| Styling | Tailwind CSS | จัดสไตล์เร็ว โทน fintech ตาม DESIGN.md |
| Database | Supabase (Postgres) | ฟรี tier + auth + API สำเร็จรูป + pgvector เผื่ออนาคต |
| Deploy | Vercel (ตอนนี้) | ฟรี ไม่ต้อง Docker |

ทิศทางอนาคต: Docker + k3s บน VPS แยก dev/prod (ออกแบบให้ config อยู่ที่ env var
ทั้งหมด จะได้ย้ายได้โดยไม่ต้องรื้อ)

## เริ่มใช้งาน

```bash
npm install
cp .env.local.example .env.local
# กรอก NEXT_PUBLIC_SUPABASE_URL และ NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
# (เอาจาก Supabase project → ปุ่ม Connect หรือ Settings → API Keys)
npm run dev
```

เปิด http://localhost:3000

### ตั้งค่า database

## โครงสร้างโปรเจกต์

```
portfolio-tracker/
├── src/
│   ├── app/                  # หน้าเว็บ (Next.js App Router)
│   │   ├── page.tsx          #   / — holdings + กำไร/ขาดทุน
│   │   ├── targets/          #   /targets — ตั้งสัดส่วนเป้าหมาย
│   │   └── rebalancing/      #   /rebalancing — เทียบปัจจุบัน vs เป้าหมาย
│   └── lib/
│       └── supabase.ts       # ตัวเชื่อมต่อ Supabase (อ่าน key จาก env)
├── migrations/               # การเปลี่ยน schema แบบเรียงลำดับ
│   ├── 0001_init.sql         #   ตารางตั้งต้น + views
│   └── 0002_add_targets.sql  #   ตาราง targets (Phase 2)
├── seed_data.sql             # ข้อมูลพอร์ตจริง (ไว้ restore, ไม่ใช่ schema)
├── .env.local               # key จริง (gitignored ไม่ขึ้น git)
├── .claude/                  # config Claude Code (hook กัน .env หลุด)
└── [เอกสารโปรเจกต์ — ดูตารางด้านล่าง]
```

### ระบบเอกสาร ("living docs")

โปรเจกต์นี้แยกเอกสารเป็นหลายไฟล์ แต่ละไฟล์เก็บคนละเรื่อง เพื่อให้อ่านเฉพาะที่
เกี่ยวกับงานตรงหน้า (ไม่ต้องโหลดทุกอย่างพร้อมกัน) — ช่วยทั้งคนและ AI ที่มาช่วย
พัฒนา

| ไฟล์ | เก็บอะไร |
|------|----------|
| `CLAUDE.md` | จุดเริ่มต้น/แผนที่ — กฎห้ามพลาด + ตารางบอกว่าไฟล์ไหนอ่านเมื่อไหร่ (บางที่สุด โหลดทุก session) |
| `ARCHITECTURE.md` | โครงสร้างระบบ — tech stack, data model, schema, การตั้งค่า Supabase, security, deployment |
| `DESIGN.md` | แนวทางหน้าตา — โทนสี, ฟอนต์, layout, กฎ UI (fintech สะอาด, กำไรเขียว/ขาดทุนแดง, dark mode) |
| `DECISIONS.md` | บันทึกการตัดสินใจทุกข้อ + "ทำไม" — กันการ revert ของที่ตัดสินใจไปแล้ว |
| `ROADMAP.md` | แผนงานแบ่งเป็น Phase 1-6 (ทำอะไรก่อน-หลัง) |
| `CHANGELOG.md` | บันทึกงานที่ทำเสร็จแล้ว เรียงตามวันที่ |
| `migrations/README.md` | กฎการทำ migration + ตารางว่ามีไฟล์ไหนทำอะไรบ้าง |

**หลักการของระบบนี้:** เอกสารระดับบน (`CLAUDE.md`, `ARCHITECTURE.md`) เขียนแบบ
เสถียร แก้น้อยที่สุด อ้างอิงแบบ "หลักการ" ไม่ผูกไฟล์ตายตัว ส่วนไฟล์ที่อัปเดต
บ่อยตามงานจริงคือ `CHANGELOG.md` (งานเสร็จ) กับ `migrations/README.md`
(schema เปลี่ยน)
